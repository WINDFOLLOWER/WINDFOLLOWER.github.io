写这个博客的原因是最近写个小程序,发现之前学过的玩意都忘了,自己看了自己几个月之前写的Android代码,有种"啊这种玩意是我写的吗"的感觉,为了避免打水漂所以决定对几个关键而且常用的点进行复习,顺便嘛,提高一下我那个只有可怜51wpm的打字速度.最后如果有机会的话可以把那个小程序给码完(Flag高高立起)
本篇主要讲的是LBS,我采用的是高德地图的SDK,我们今天要达到的目标主要是获取当地的经纬度和位置信息,完成这个就ok啦!
首先使用高德的定位服务需要注册为高德的开发者,网站:高德开发者网址
然后我们打开自己的个人中心,点击我的应用->创建新应用.

在这里我们需要填写自己的程序的各种信息,具体的安全码和PackageName获取方法上面都给你了,具体可以查看他们给你的教程.这里填写的相关的东西务一定不要出错,如果填错的话和你的应用信息不一致将不会传回位置信息.

¶配置工程

在工程中配置高德SDK官方也有相应的教程:官方配置工程教程,我这里推荐的是使用Gradle集成SDK(都8012年了还手动导入jar包),打开下图工程中的build.gradle文件

在build.gradle文件的dependencies的括号内需要根据我们的需要引入对应的代码

SDK 引入代码
3D地图 implementation ‘com.amap.api:3dmap:latest.integration’
2D地图 implementation ‘com.amap.api:map2d:latest.integration’
导航 implementation ‘com.amap.api:navi-3dmap:latest.integration’
搜索 implementation ‘com.amap.api:search:latest.integration’
定位 implementation ‘com.amap.api:location:latest.integration’

这里与官方唯一的区别在于使用的是implementation而非compile,原因在于3.0以上的AS中compile被标为过时.具体可以参考这篇文章android gradle tools 3.X 中依赖，implement、api 指令

由于今天我们要使用的只有定位只需要加入表中最后一项就行,当然你全加了也没没啥问题.然后点击AS代码框上方的Sync Now蓝色字AS就会自动下载并导入相关的jar包,到此为止配置已完成

¶请求权限

为了进一步保护用户的手机安全,6.0以后权限的请求更加严格,具体可以参考这篇博客Android 6.0开始动态请求权限,我们的手机权限分为正常权限和敏感权限,正常权限只需要在工程中AndroidManifest.xml文件的manifest代码块中加入类似的这样的代码即可获得权限(当然别写在application代码块中)

而敏感权限不仅需要和正常权限在manifest中声明,还需要动态请求才能正式获得权限.
为了定位程序正常运行,我们要在manifest代码块将所有需要的权限都声明一遍

这是官方给的权限代码实际上定位不需要这么多,可以去掉蓝牙什么的权限
然后我们对敏感权限进行动态申请
在MianActivity的onCreate中加入如下代码

@Override
public void onRequestPermissionsResult(int requestCode, String[] permissions, int[] grantResults) {
//requestCode之前标明为1,如果收到的请求码为1就进行处理
switch (requestCode) {
case 1:
if (grantResults.length > 0) {
//检查每一项是否被授权
for (int result : grantResults) {
if (result != PackageManager.PERMISSION_GRANTED) {
Toast.makeText(this,“必须所有权限才能使用”, Toast.LENGTH_SHORT).show();
//无情退出2333
finish();
return;
}
//如果没有无情退出就请求位置,这个函数后续再写
requestLocation();
}
}
break;
default:break;
}
}

if (!permissionList.isEmpty()) {
String[] permissions = permissionList.toArray(new String[permissionList.size()]);
ActivityCompat.requestPermissions(MainActivity.this, permissions, 1);
} else {
//直接请求
requestLocation();
}

//首先声明一个public的Client
//我们需要使用其startLocation开始定位
//在开始定位前需要对其进行设定
public AMapLocationClient mLocationClient = null;

public void requestLocation() {
initMap(getApplicationContext());//初始化
mLocationClient.startLocation();//开始定位
}

private void initMap(Context context) {
mLocationClient = new AMapLocationClient(context);//实例化Client
//实例化Option,来设置定位
mLocationClient.setLocationListener(new mLocationListener());
AMapLocationClientOption mLocationOption = new AMapLocationClientOption();
//设置定位模式为高精度
mLocationOption.setLocationMode(AMapLocationClientOption.AMapLocationMode
.Hight_Accuracy);
//是否返回地址信息
mLocationOption.setNeedAddress(true);
//设置Option,通过Option传入各项设置
mLocationClient.setLocationOption(mLocationOption);
}
